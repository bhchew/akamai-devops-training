Gotchas

Beware of TIME

  • Clock skew of at most 30 seconds
  • Otherwise 400 response code and Invalid timestamp is returned
  • Advised to use NTP service on production machines

Beware of max-body

  • Ensure max-body in your .edgerc file is not lower than maximum size of the payload
  • Otherwise 400 response code and Signature mismatch is returned

Beware of Encoding

Some editors add the BOM sequence and this is not expected by OPEN API.

https://en.wikipedia.org/wiki/Byte_order_mark

Using Docker

A docker image is provided with all official commands pre-installed:

docker run --rm -it akamaiopen/cli <command args>

Making it shorter

function akamai {
     if [[ `docker ps | grep akamai-cli$ | wc -l` -eq 1 ]]; then
         docker exec -it akamai-cli akamai $@;
     elif docker start akamai-cli > /dev/null 2>&1 && sleep 3 && docker exec -it akamai-cli akamai $@; then
         return 0;
     else
         echo "Creating new docker container"
         docker create -it -v $HOME/.edgerc:/root/.edgerc --name akamai-cli akamai/cli > /dev/null 2>&1 && akamai $@;
     fi;
}

Initializing the CLI

We need to create the following file:

cat ~/.akamai-cli/config
[cli]
cache-path            = /Users/me/.akamai-cli/cache
config-version        = 1
last-upgrade-check    = ignore
install-in-path       =
client-id             = xxxxxx-xxxx-xxx-xxxx-xxxxxxxxxx
enable-cli-statistics = false
last-ping             = 2018-10-17T15:15:02+02:00

Initializing the CLI (Cntd.)

cat <<EOF > ~/.akamai-cli/config
[cli]
cache-path            = ~/.akamai-cli/cache
config-version        = 1
enable-cli-statistics = false
last-ping             = $(date -u "+%Y-%M-%DT%TZ")
client-id             =
install-in-path       =
last-upgrade-check    = ignore
EOF

Python & Flask

@app.route('/activate', methods=['POST'])
def activate():
    content = request.get_json(silent=True)
    rules = build_rules(content)
    data = {
        "description": "created via OPEN API",
        "matchRuleFormat": "1.0",
        "matchRules": rules
    }
    result = edge_session.post(
        urlparse.urljoin(api_baseurl, '/cloudlets/api/v2/policies/%s/versions' % POLICY_ID),
        data=json.dumps(data), headers=api_headers)

    if result.status_code != 201:
        app.logger.exception(result.content)
        abort(503)

    version = json.loads(result.content)['version']
    app.logger.info("Policy updated, version=%s" % version)

    data = {"network": "prod"}

    result = edge_session.post(
        urlparse.urljoin(api_baseurl,
                         '/cloudlets/api/v2/policies/%s/versions/%s/activations' % (POLICY_ID, version)),
        data=json.dumps(data), headers=api_headers)

    if result.status_code != 200:
        app.logger.exception(result.content)
        abort(503)

    app.logger.info("Policy activated, version=%s" % version)

    return result.content

Akamai Sandbox

Increase development velocity and confidence through Isolated environments for testing property changes before deployment

Formerly known as DevPops API, GA June 2019

  • Step 1: Request is received by Sandbox client and proxied over a websocket connection to the Sandbox network infrastructure
  • Step 2: Request is processed by an Akamai server and forwarded back to the local origin, through the Sandbox Client
  • Step 3: Response is generated by the local origin and forwarded back to the Akamai server, through the Sandbox Client

Sandbox Components

  1. Sandbox API
    Low level interface to the Sandbox system, useful for custom integrations
  2. Sandbox CLI
    Easy-to-use command-line module for Akamai CLI

Sandbox CLI is the way to go for most use cases.

What does it support?

  • Most Delivery Products are supported
  • Most behaviors are supported
  • Cloudlets and Image Manager will be processed according to staging policy
  • Web Application Firewall rules are NOT applied
